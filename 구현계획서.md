# 🏫 교우관계 조사 및 분석 프로그램 구현 계획서

> **대상**: 3학년 2반 (총 11명)  
> **목적**: 학급 내 교우관계를 체계적으로 조사하고 시각적으로 분석하여 학급 운영에 활용  
> **기술 스택**: Node.js + Express 로컬 서버 + HTML/CSS/JS  
> **작성일**: 2026년 3월 26일

---

## 📋 프로그램 개요

### 핵심 기능
1. **설문조사 시스템** — 학생들이 **QR코드**를 스캔하여 태블릿/스마트폰에서 설문 응답
2. **결과 분석 대시보드** — 교사 전용 시각적 분석 도구 (비밀번호 보호)
3. **자동 파일 저장** — 학생 응답이 교사 컴퓨터의 `data/` 폴더에 JSON 파일로 자동 저장

### 운영 방식

```
┌──────────────────────────────────────────────────────────┐
│                    같은 WiFi 네트워크                       │
│                                                          │
│   📱 학생 태블릿/폰                  💻 교사 컴퓨터         │
│   ┌──────────┐                     ┌──────────────┐     │
│   │ QR 스캔   │ ──── HTTP ────▶   │ Node.js 서버  │     │
│   │ 설문 응답  │                    │ (Express)    │     │
│   └──────────┘                     │              │     │
│   📱 📱 📱 📱 📱                    │  data/       │     │
│   (11명 동시 접속)                   │  ├─ 1_김소미.json│   │
│                                    │  ├─ 2_김철호.json│   │
│                                    │  └─ ...       │     │
│                                    └──────────────┘     │
│                                         │               │
│                                    🖥️ 분석 대시보드       │
│                                    (교사 전용, 비밀번호)    │
└──────────────────────────────────────────────────────────┘
```

| 항목 | 선택 | 이유 |
|------|------|------|

## 🚀 서버 구성

### 서버 시작 흐름
1. 교사가 `start.bat`을 더블클릭
2. Node.js 서버가 자동 시작 (포트 3000)
3. 콘솔에 **접속 URL**과 **QR코드** 자동 표시
4. 교사가 QR코드를 칠판/TV에 띄워줌
5. 학생들이 QR코드를 스캔하여 설문 페이지 접속

### API 엔드포인트

| 메서드 | 경로 | 설명 |
|--------|------|------|
| `GET` | `/` | 학생용 설문 페이지 |
| `GET` | `/admin` | 교사용 분석 대시보드 (비밀번호 필요) |
| `POST` | `/api/survey` | 설문 응답 저장 (→ JSON 파일) |
| `GET` | `/api/survey` | 모든 응답 데이터 조회 (분석용) |
| `GET` | `/api/survey/:id` | 특정 학생 응답 조회 |
| `GET` | `/api/status` | 응답 현황 (몇 명 완료했는지) |
| `DELETE` | `/api/survey/:id` | 특정 학생 응답 삭제 (재설문) |

### 서버 실행 배치 파일 (`start.bat`)
```batch
@echo off
chcp 65001 > nul
title 교우관계 조사 서버

echo.
echo  ╔══════════════════════════════════════╗
echo  ║   🏫 교우관계 조사 프로그램 서버      ║
echo  ║   3학년 2반                          ║
echo  ╚══════════════════════════════════════╝
echo.

:: Node.js 설치 확인
where node >nul 2>nul
if errorlevel 1 (
    echo  ❌ Node.js가 설치되어 있지 않습니다!
    echo  https://nodejs.org 에서 Node.js를 설치해주세요.
    pause
    exit /b
)

:: node_modules 없으면 npm install 실행
if not exist "node_modules" (
    echo  📦 처음 실행: 필요한 패키지를 설치합니다...
    npm install
    echo.
)

:: data 폴더 없으면 생성
if not exist "data" mkdir data

echo  🚀 서버를 시작합니다...
echo.
node server.js

pause
```

### 서버 종료 배치 파일 (`stop.bat`)
```batch
@echo off
chcp 65001 > nul
echo 교우관계 조사 서버를 종료합니다...
taskkill /f /im node.exe 2>nul
echo 서버가 종료되었습니다.
pause
```

## 📊 설문 문항 구성 (최신 소시오메트리 기법 반영)

### 설문 설계 원칙
- **또래 지명법(Peer Nomination)**: 긍정/부정 양방향 지명 포함
- **또래 평정법(Peer Rating)**: 리커트 척도 체크 문항
- **'우리반 관계 읽기' AI 플랫폼 구조 참고**: 3단계(나의 마음 → 친구 관계 → 학급 전체) 구조 적용
- 학생 선택은 **클릭 선택 버튼** 방식 (직접 입력 X, 자기 이름 제외)

### 설문 응답 흐름

```
1단계: 본인 이름 선택
  11명 중 자기 이름을 클릭
  → 이미 응답 완료한 학생은 비활성화 표시
       │
       ▼
2단계: 문항 1~12 (지명법)
  자기 이름이 제외된 나머지 10명 중에서 선택 (최대 3명)
  + 이유 입력
       │
       ▼
3단계: 체크 문항 C1~C4 (리커트 척도)
  5점 척도 선택
       │
       ▼
4단계: 제출 완료
  서버에 자동 저장 → 완료 메시지
```

### 문항 구성 (총 3파트, 12문항 + 체크 4문항)

#### 🟢 파트 1: 긍정적 관계 (지명법)
> 친한 친구, 신뢰, 협동 관계를 파악

| # | 문항 | 유형 | 최대 선택 |
|---|------|------|-----------|
| 1 | 쉬는 시간에 같이 놀고 싶은 친구는? | 다중 선택 + 이유 | 3명 |
| 2 | 모둠 활동을 할 때 함께하면 좋을 것 같은 친구는? | 다중 선택 + 이유 | 3명 |
| 3 | 내가 힘들거나 속상할 때 이야기하고 싶은 친구는? | 다중 선택 + 이유 | 3명 |
| 4 | 공부하거나 활동할 때 도움을 잘 주는 친구는? | 다중 선택 + 이유 | 3명 |
| 5 | 나를 잘 이해해 준다고 느끼는 친구는? | 다중 선택 + 이유 | 3명 |

#### 🔵 파트 2: 관계 변화 및 학급 분위기 (지명법)
> 관계 역동성과 사회적 역할 파악

| # | 문항 | 유형 | 최대 선택 |
|---|------|------|-----------|
| 6 | 요즘 더 자주 같이 지내는 친구는? | 다중 선택 + 이유 | 3명 |
| 7 | 예전보다 더 친해졌다고 느끼는 친구는? | 다중 선택 + 이유 | 3명 |
| 8 | 우리 반에서 친구들에게 친절하게 잘 대해 주는 친구는? | 다중 선택 + 이유 | 3명 |
| 9 | 우리 반에서 두루두루 잘 어울리는 친구는? | 다중 선택 + 이유 | 3명 |

#### 🟠 파트 3: 관계 어려움 (지명법, 비공개 강조)
> 갈등 관계 및 교사 개입 필요 영역 파악

| # | 문항 | 유형 | 최대 선택 |
|---|------|------|-----------|
| 10 | 함께 활동할 때 조금 불편하거나 어색한 친구가 있나요? | 다중 선택 + 이유 | 3명 |
| 11 | 같이 있으면 다투거나 의견이 잘 안 맞는 친구가 있나요? | 다중 선택 + 이유 | 3명 |
| 12 | 선생님이 더 살펴보면 좋겠다고 생각하는 친구관계가 있나요? | 자유 서술 | - |

#### ✅ 파트 4: 자기 인식 체크 (리커트 5점 척도)
> 학급 내 소속감, 안전감 측정

| # | 문항 | 척도 |
|---|------|------|
| C1 | 나는 우리 반에서 함께 놀 친구가 있다 | 5점 (아주 그렇다 ~ 전혀 그렇지 않다) |
| C2 | 나는 우리 반에서 힘들 때 도움을 청할 친구가 있다 | 5점 |
| C3 | 나는 우리 반에서 대체로 편안함을 느낀다 | 5점 |
| C4 | 나는 우리 반 친구들과 잘 어울리는 편이다 | 5점 |

## 🔍 분석 기능 상세

### 1. 소시오그램 (네트워크 그래프)
> D3.js Force-Directed Graph 활용

- **노드(학생)**: 원형, 이름 표시, 성별 색상 구분
- **노드 크기**: 지명 받은 횟수(인기도)에 비례
- **엣지(관계선)**:
  - 🟢 초록 실선: 긍정적 지명 (문항 1~9)
  - 🟠 주황 점선: 부정적 지명 (문항 10~11)
  - 선 두께: 지명 빈도(여러 문항에서 중복 지명 시 두꺼움)
  - 화살표 방향: 지명한 방향 표시
- **상호 지명**: 양방향 화살표로 표시 (쌍방 친구)
- **필터링**: 문항별/유형별 관계선 ON/OFF 토글
- **인터랙션**: 노드 클릭 시 해당 학생 관계 하이라이트

### 2. 관계 유형 분석
| 유형 | 설명 |
|------|------|
| ⭐ 인기아(Popular) | 긍정 지명 많이 받은 학생 |
| 👻 고립아(Isolated) | 지명을 거의 받지 못한 학생 |
| ⚠️ 거부아(Rejected) | 부정 지명이 많은 학생 |
| 🔄 양면아(Controversial) | 긍정+부정 지명 모두 많은 학생 |
| 🌉 중심 연결자(Connector) | 다리 역할을 하는 학생 (문항 9 기반) |
| 🤝 상호 선택(Mutual) | 서로 지명한 쌍 |

### 3. 관계 행렬 (히트맵)
- 11×11 매트릭스 형태
- 행: 지명한 학생, 열: 지명받은 학생
- 셀 색상 강도: 지명 빈도
- 긍정/부정 분리 보기

### 4. 개인 프로필 카드
- 학생별 상세 분석 카드
- 받은 지명 수 (긍정/부정 분리)
- 한 지명 수
- 상호 선택 친구 목록
- 체크 문항 응답 결과
- **관심 필요 알림**: 고립/거부 위험 표시

### 5. 학급 전체 통계
- 체크 문항 결과 레이더 차트 (학급 평균)
- 긍정/부정 지명 분포 막대 차트
- 성별 교차 관계 분석
- 소집단(클리크) 감지

### 6. 데이터 관리
- JSON 파일 자동 저장 (학생별 개별 파일)
- 전체 데이터 JSON 내보내기
- 분석 결과 인쇄용 화면

## 🎯 분석 지표 계산 방법

### 관계 유형 분류 기준
| 유형 | 조건 |
|------|------|
| 인기아 (⭐ Popular) | 긍정 지명 상위 25% AND 부정 지명 하위 50% |
| 거부아 (⚠️ Rejected) | 부정 지명 상위 25% AND 긍정 지명 하위 50% |
| 양면아 (🔄 Controversial) | 긍정 + 부정 지명 모두 상위 50% |
| 무시아 (👻 Neglected) | 긍정 + 부정 지명 모두 하위 25% |
| 보통 (✅ Average) | 위 조건에 해당하지 않음 |

### 관심 필요 학생 알림 조건
- 긍정 지명 0~1회
- 부정 지명 3회 이상
- 상호 선택 친구 0명
- 체크 문항 평균 2.0 이하

## 🔧 구현 단계

### Phase 1: 서버 + 기반 구축
1. `package.json` — 프로젝트 설정 및 의존성 (express, qrcode)
2. `server.js` — Express 서버 + REST API + QR코드 콘솔 표시
3. `start.bat` / `stop.bat` — 서버 실행/종료 배치 파일
4. `data.js` — 학생 목록(11명), 문항 데이터 정의

### Phase 2: 학생용 설문 시스템
5. `survey.html` — 학생용 설문 페이지 (모바일 최적화)
6. `style.css` — 디자인 시스템 (파스텔톤, 모바일 우선 반응형)
7. `survey.js` — 설문 로직
   - **1단계**: 11명 중 본인 이름 선택 (이미 응답 완료한 학생은 ✅ 비활성화)
   - **2단계**: 문항별 친구 선택 UI (토글 버튼, 자기 이름 제외, 최대 3명)
   - **3단계**: 이유 입력, 체크 문항 응답
   - **4단계**: 설문 완료 → `POST /api/survey`로 서버에 전송

### Phase 3: 분석 엔진
8. `analysis.js` — 핵심 분석 알고리즘
   - 지명 빈도 계산
   - 상호 선택 감지
   - 관계 유형 분류 (인기아, 고립아, 거부아 등)
   - 소집단 감지
   - 체크 문항 통계

### Phase 4: 교사 대시보드 + 시각화
9. `admin.html` — 교사용 분석 페이지 (비밀번호 보호)
10. `admin.js` — 대시보드 로직 (데이터 로드 → 시각화)
11. `sociogram.js` — D3.js 소시오그램
12. `charts.js` — Chart.js 차트 + 히트맵 + 개인 프로필 카드

### Phase 5: 마무리
13. 응답 현황 실시간 표시
14. 데이터 내보내기 기능
15. 인쇄 스타일 최적화
16. 최종 테스트 및 폴리시

## ⚡ 보안 및 운영 고려사항

1. **개인정보 보호**: 모든 데이터는 교사 PC 로컬에만 저장 (인터넷 전송 없음)
2. **응답 비밀 보장**: 학생은 자기 응답만 작성, 다른 학생 응답 확인 불가
3. **교사 전용 분석**: `/admin` 페이지는 비밀번호 보호
4. **중복 응답 방지**: 이미 응답한 학생 이름은 비활성화 처리
5. **재설문 지원**: 교사가 특정 학생 응답을 삭제하고 재작성 가능
6. **데이터 영구 보존**: JSON 파일로 저장되어 서버 종료 후에도 유지
7. **같은 WiFi 필수**: 교사 PC와 학생 기기가 같은 네트워크에 연결 필요

## 📋 사전 준비사항 (교사)

### Node.js 설치 (한 번만)
1. https://nodejs.org 에서 LTS 버전 다운로드
2. 설치 파일 실행 → 기본 설정으로 설치
3. 설치 확인: 명령 프롬프트에서 `node --version` 입력

### 사용 방법
1. `start.bat` 더블클릭 → 서버 시작
2. 콘솔에 표시되는 QR코드를 TV/칠판에 보여줌
3. 학생들이 QR코드 스캔 → 자기 이름 선택 → 설문 작성
4. 교사는 `http://내IP:3000/admin` 접속 → 분석 확인
5. 사용 후 `stop.bat` 또는 콘솔 창 닫기로 서버 종료
